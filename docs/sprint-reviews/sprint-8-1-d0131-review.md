## 4. 真实发现（最有价值的部分）

### 发现 1：DMA 是转折点指标，不是趋势强度指标

**现象**：测试断言 `DMA > 0 = 多头信号` → 失败。

**根因分析**：
- 公式：`DMA = DDD - 2*AMA`
- 稳态下：`AMA ≈ DDD`（AMA 是 DDD 的 10 日平滑）
- 推论：`DMA ≈ DDD - 2*DDD = -DDD` → 稳态恒负
- 真信号：DDD 由负转正（多头转折）/ 由正转负（空头转折）

**修复**：测试从"断言 DMA 符号"改为"断言转折点后 DDD 方向"。

**业务含义**：DMA 在 trend_up 加权中应配合其他趋势因子使用，不宜单独高权重（30% 偏高，建议 D-013.5 调低到 15~20%）。

### 发现 2：D-013 命名分歧债

**现象**：daily 跑分 KeyError "FactorSet 因子 {T6_dma, T7_ma_arrangement} 在 WeightScheme 中无权重"。

**根因**：
- D-004 B2 方案（2026-06-25）配置用通用名 `DMA / FIB`
- v2 FACTOR_REGISTRY（2026-06-21）用内部 ID `T6_dma / T7_ma_arrangement`
- D-013 Sprint-8 设计时没真接入 B2 权重到 daily 跑分链

**修复**：D-013.1 同步两边命名（WeightScheme + JSON 配置）。

**教训**：Sprint 设计必须有"端到端跑分验证"环节，不然命名债不会暴露。

### 发现 3：测试数据要带转折点

**现象**：稳态线性趋势数据 `close = 100 + n*2.0` 下，DMA 输出恒为 -0.07（无变化）。

**根因**：完美线性数据下 MA10/MA50 斜率相同，DDD 趋近常量。

**修复**：测试数据改为"前 50 日横盘 + 后 150 日趋势"（带转折点）。

**教训**：技术指标测试必须用"带转折点"数据（前段横盘 + 后段趋势），否则测不出信号。
