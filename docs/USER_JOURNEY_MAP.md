## 0. 一句话定位（结论先行）

> **v2 的核心是 1 条主路径**：**散户从 0 到 1 学会建立自己的量化模型**。
>
> 1 个旁支（研究者 CLI）服务于不同细分场景，但**不与主路径竞争设计资源**。
>
> 产品形态：**CLI（研究者）+ Web（散户）双形态**。
>
> 产品本质：**量化教育产品**，不是"投资工具"。

### 0.1 3 类用户的产品决策

| 角色 | 决策 | 理由 |
|---|---|---|
| 散户学习者（A+B） | **主路径，唯一重点优化对象** | Q1=A+B，最大用户群 |
| 研究者（5 年 Python） | 旁支 B，**CLI 模式，不做 Web** | 服务种子用户/协作者 |
| AI Agent | 不是产品用户，**只走 QwenPaw 触发词集成** | 技术集成而非产品用户 |

> **历史决策（已删除）**：原"散户极简版（只要 BUY 信号）"作为旁支 A 已被删除。详见决策 #015。

### 1.2 4 做了的

1. **按"主路径 + 旁支"画地图**——不是 4 类用户并列
2. **统一节奏**——主路径每阶段 4 步（原理→默认→调整→验证）
3. **触发词命名避免技术词**——learn / quick / docs，不是 alpha/beta/universe
4. **诚实标注**——每个阶段标注"v2 已支持 vs 还缺什么"

### 1.3 4 没做的（**含反向风险评估**）

| # | 没做的 | 理由 | **反向风险** | 缓解措施 |
|---|---|---|---|---|
| 1 | ❌ 不做"投资工具"宣传 | 监管 + 定位冲突 | 低（监管必须）| — |
| 2 | ❌ 不让散户**写**代码 | 教育型定位 + 降低门槛 | **中（损失"我也能改"的成就感）** | **阶段 5 毕业可自愿进入"只读代码"模式**（只读，不写；满足好奇心，不增加负担） |
| 3 | ❌ 不做实时行情订阅 | 散户要"日频"，不需要 tick | 低（明确取舍）| — |
| 4 | ❌ 不做券商交易接口 | 监管复杂 + 与教育定位冲突 | 低（监管必须）| — |

**反向风险评估结论**：4 项中 3 项低风险，**仅第 2 项有中风险**，通过"只读代码"旁门缓解。

## 3. 主路径：散户从 0 到 1 入门（5 阶段 × 4 步）

### 阶段 0：定位（3 句话分流）

```
输入：用户进入 /learn
  ↓
问 3 句话（Web 表单）
  Q1. "你想学量化吗？"
  Q2. "你能每周投入 30 分钟吗？"
  Q3. "你能接受'暂时不能交易'吗？"（强调教育定位）
  ↓
判定（≥2 答是）→ 进入主路径
判定（<2 答是）→ 提示"本产品是教育产品，建议先学习再使用"（不跳转，不流失）
```

**业务对象**：`LearnerProfile`（3 个布尔字段 + 完成时间戳）

**设计变更说明**：原设计"<2 答是 → 跳到旁支 A"已删除（旁支 A 不存在）。<2 答是的用户**不是产品目标用户**，应明确告知而非提供受限路径。

### 阶段 1：择股（建立 ETF 池）

```
第 1 步：原理（≥100 字）
  - 散户问"什么是择股"
  - Web 解释：从 1486 只 ETF 筛选 → 4 个条件
    ①流动性（日均成交额）  ②盘子（规模）  ③跟踪误差  ④年限
  ↓
第 2 步：v2 默认
  - Web 展示：14 核心 + 300 参考（图文卡片）
  ↓
第 3 步：调整
  - Web 表单：加 X / 去 Y（自动解释为什么合理）
  - 例子："加 510300？✅ 这是沪深 300 大盘，加它合理"
  ↓
第 4 步：验证
  - Web 跑池子统计（核心/参考分布 + 4 条件覆盖）
  - 显示覆盖率 0-100%
```

**业务对象**：`ETFPool`（code/name/pool_role=core/reference/excluded）

### 阶段 2：择时（多因子组合）

```
第 1 步：什么是因子
  - Web 解释：3 类（入场/过滤/出场）
  - 配图：29 因子分布饼图（趋势 T1~T7 / 动量 M1~M6 / 量能 V1~V4 / 强度 S1~S2 / 反转 N1~N3）
  ↓
第 2 步：C21-1 + 29 因子（D-013.1 Sprint-8.1 补 T6_dma + T7_ma_arrangement）
  - Web 解释 C21-1：BOLL 中轨 + MA60（v1 金三角）
  - 显示当前默认因子组合
  ↓
第 3 步：加自己的因子
  - Web 选 1-2 个新因子（系统引导："选哪个？解释什么含义？"）
  - 系统自动注册 + 跑 4 验证器
  ↓
第 4 步：看综合分
  - Web 跑 ComprehensiveValidator
  - 显示 4 个分项：Walk Forward / Monte Carlo / Cross ETF / Consistency
  - 综合分 ≥ 0.6 才算通过
```

**业务对象**：`CompositeScore`（composite_score + 4 个验证器分项）

### 阶段 3：仓位管理（v2 已支持 / v3 占位）

> **当前状态**：v2.0 已实现完整仓位管理（PositionGuide 22 字段 + 9 步决策树）。
> 散户 0 代码使用：Web UI 调 4-6 个关键参数；旁支 B 调 22 全字段。
> 详细业务对象：见 Step 5 完成后回写（依赖 MODULES.md 的 risk 模块文档）。

### 阶段 4：日循环（习惯回路 + cron 触发）✅ v3.0 已设计

> **核心机制**：借用 Atomic Habits 4 步回路（提示/渴求/反应/奖赏），让散户"每天 30 秒就够"。

```
第 1 步：原理（≥100 字）
  - 散户问"为什么每天要打开"
  - Web 解释：4 步回路（Atomic Habits）
    ① 提示（Cue）     → 14:30 工作日 cron 推送 + Web 通知
    ② 渴求（Craving） → 持仓检查 + 信号识别（2 分钟看完）
    ③ 反应（Response）→ 自动跑 1 次 daily 决策（30 秒）
    ④ 奖赏（Reward）  → 段位分 +1 + 报告推送（钉钉/邮件）
  ↓
第 2 步：v2 默认
  - Web 展示：每天 14:30 自动跑（cron 触发 run_daily.py）
  - 散户无需手动操作（仅"查看报告"）
  ↓
第 3 步：调整
  - Web 表单：改触发时间 / 加额外提醒 / 关掉某类推送
  - 例子："加 09:30 早盘提醒？✅ 帮助开盘前看隔夜美股影响"
  ↓
第 4 步：验证
  - Web 跑日循环统计（近 7 天 / 30 天推送成功率 + 点击率）
  - 显示漏斗：推送 100% → 点击 60% → 行动 30%
```

**业务对象**：`DailyLoop`（cron_schedule + 4 步状态 + 段位分 + 漏斗统计）

**业界参考**：
- Atomic Habits 4 步回路（提示/渴求/反应/奖赏）
- Shape Up Ch 8 "Betting Table"（6 周 cycle → 缩到日级 cycle）
- Stripe Docs "30 seconds to first API call"（最小可运行示例）

**Appetite**（按 Shape Up）：小（0.3 人天，cron 09:30 + 14:30 已计划 ROADMAP Sprint 2.2）

## 7. 异常处理（8 类完整版）

> **从 v2.0 的 3 类扩展到 v3.0 的 8 类**——按数据/系统/用户行为 3 大类分。

### 7.1 数据类（4 类）

| # | 异常 | 症状（用户看到什么） | 自救（用户能做什么） | AI 何时介入 |
|---|---|---|---|---|
| 1 | **数据过期** | "最后更新：3 天前" | 等第二天 14:30 cron；或问 AI "怎么刷新" | AI 解释 cron 机制 |
| 2 | **数据缺失** | "ETF 159999 无 K 线" | 换 ETF；或问 AI "为什么没有" | AI 检查数据源 |
| 3 | **数据异常** | "某日涨幅 -50%" | 不操作；或问 AI "这是真实还是 bug" | AI 判断 NaN/Inf/真实跌停 |
| 4 | **因子失败** | "IC 计算抛 ValueError" | 换因子；或问 AI "为什么失败" | AI 看 traceback + 因子定义 |

### 7.2 系统类（2 类）

| # | 异常 | 症状 | 自救 | AI 何时介入 |
|---|---|---|---|---|
| 5 | **API 限流** | "请求被拒，请等待" | 等待 5 秒重试 | AI 解释 AKTools 5s 规则 |
| 6 | **数据库锁** | "database is locked" | 等待 30 秒；或问 AI "怎么解锁" | AI 触发 WAL 模式检查 |

### 7.3 用户行为类（2 类）

| # | 异常 | 症状 | 自救 | AI 何时介入 |
|---|---|---|---|---|
| 7 | **路径中断** | 用户关闭浏览器 / 网络断 | 重启后自动恢复（state.json） | AI 询问"继续 / 重来 / 切换" |
| 8 | **配置冲突** | "止损 < 止盈，请重设" | 按提示改参数 | AI 解释为什么冲突（如 5% < 10%） |

## 10. 30 秒找入口（✅ v3.0 已设计）

> **设计目标**（按 Stripe Docs "30 seconds to first API call"）：新用户 30 秒内跑通最小路径。
> **核心原则**：每个入口对应 1 个命令（不是 1 段文档），散户 0 文档。

### 10.1 散户主路径（4 命令 30 秒）

|:---:|---|:---:|---|
| 1. 定位 | `learn` | 5s | 进入 3 句话分流（Q1/Q2/Q3） |
| 2. 择股 | `etf` | 5s | 看到 14 核心 + 300 参考（图文卡片） |
| 3. 因子 | `factor` | 10s | 看到 29 因子分布 + 当前默认组合 |
| 4. 验证 | `check` | 10s | 跑 ComprehensiveValidator 4 验证器（综合分 ≥ 0.6） |
