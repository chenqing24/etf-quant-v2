## 1. 调研流程

```
1. 测试所有候选数据源（不只是想用的）
2. 验证返回格式（字段顺序、类型）
3. 写入 docs/DATA_SOURCE_REFERENCE.md
4. 部署：限流严重时部署本地服务（aktools-server）
5. 测试：先小批量再全量
6. 集成：统一 HTTP API 入口
```

## 3. AKTools HTTP API 规范

```
服务地址：http://127.0.0.1:8080
启动命令：cd aktools-server && python -m aktools
调用规则：间隔≥5秒，避免并发

可用接口：
- /api/public/fund_etf_spot_em     # 1486条全市场ETF，16秒
- /api/public/fund_etf_hist_sina    # 单只ETF历史3400条，0.24秒
- /api/public/fund_etf_scale_sse    # 上交所ETF规模593条，0.5秒
```

## 4. 数据源可靠性排序

```
高 → 低
腾讯API > 新浪API > AKShare新浪 > 天天基金 > BaoStock > AKShare东财
```

## 5. 限速保护

- 任何数据源调用前先 sleep
- 大接口（如 fund_etf_spot_em）间隔 10 秒+
- 错误重试用指数退避（2/4/8/16 秒）

## 6. 数据源不可用时

- 自动降级到下一级数据源
- 写入 DATA_SOURCE_REFERENCE.md 标注不可用
- 通知用户（notify 模块）
